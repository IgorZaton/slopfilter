# Chrome Web Store

**Fast path:** follow **[QUICK_PUBLISH.md](./QUICK_PUBLISH.md)** (~30 min of your time + 1–3 day review).

This guide covers publishing **SlopFilter** to the Chrome Web Store.

## Prerequisites

1. [Chrome Web Store developer account](https://chrome.google.com/webstore/devconsole) ($5 one-time).
2. Node.js 18+ and `npm install` in this repo.
3. A hosted **privacy policy** URL:

   `https://igorzaton.github.io/slopfilter/privacy-policy.html`

   (deployed from [`docs/`](./) via GitHub Actions — see [Pages setup](#github-pages-setup) below).

## Build the upload ZIP

```bash
npm install
npm run store:assets
```

This runs the ML bundle, generates icons, stages `dist/slopfilter-<version>/`, and writes:

```text
dist/slopfilter-<version>.zip   ← upload this file
```

Verify locally before uploading:

1. `chrome://extensions` → Developer mode → **Load unpacked** → select `dist/slopfilter-<version>/`
2. Open a supported social site and confirm classification works (first run downloads ~100 MB model).

### Permission justifications (review form)

| Permission | Justification |
|------------|----------------|
| `storage` | Save user settings and scan statistics shown in the popup. |
| `offscreen` | Run ONNX inference in a persistent document (MV3 service workers cannot host WASM ML). |
| Social / forum hosts (listed in manifest) | Inject content script to read posts and apply dim/hide/badge on supported sites only. |
| `huggingface.co` / `cdn-lfs.huggingface.co` | One-time download of open-source ONNX model weights; cached locally. No user post text is sent. |

### Single purpose

> Help users identify and visually de-emphasize likely AI-generated social posts using on-device classification.

## GitHub Pages setup

If `https://igorzaton.github.io/slopfilter/` is **404**, Pages is not live yet:

1. Open [slopfilter → Settings → Pages](https://github.com/IgorZaton/slopfilter/settings/pages).
2. Under **Build and deployment → Source**, choose **GitHub Actions** (not “Deploy from a branch”).
3. Open [Actions](https://github.com/IgorZaton/slopfilter/actions) → **Deploy privacy policy to Pages** → **Run workflow** → Run.
4. Wait until the run is green; the site URL appears in the job summary and under Settings → Pages.
5. Confirm in a private window:
   - `https://igorzaton.github.io/slopfilter/` → privacy policy
   - `https://igorzaton.github.io/slopfilter/privacy-policy.html` → same page

The workflow only auto-runs on pushes to `main` that touch `docs/**` or `.github/workflows/pages.yml`.

## Versioning

Chrome requires `manifest.version` to be **dot-separated integers** (e.g. `1.0.0`, `1.0.1`).

Bump `version` in `manifest.json` for each store upload.

## Scripts reference

| Script | Purpose |
|--------|---------|
| `npm run build:ml` | Copy offscreen ONNX runtime + WASM to `vendor/` |
| `npm run build:icons` | Render `icons/icon-*.png` from `icons/myicon.png` |
| `npm run build` | Both of the above |
| `npm run package` | `build` + store upload ZIP |
